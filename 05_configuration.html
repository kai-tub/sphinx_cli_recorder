Configuration#

Recorder Configuration#

Columns#

field AsciinemaRecorderSettings.cols: pydantic.types.PositiveInt = 80#

Define the number of columns the terminal should have. The extension will ensure that the same number of columns is used for the recording and playback. If the number of columns is too low, the output could be wrapped weirdly. The text will also be zoomed-in to fit the width of the container.

Constraints

  • exclusiveMinimum = 0

:cols: 20
:cols: 40

Rows#

field AsciinemaRecorderSettings.rows: pydantic.types.PositiveInt = 20#

Define the number of rows the terminal should have. The extension will ensure that the same number of rows is used for the recording and playback. If the number of rows is too low, the beginning of the output may be lost. Similar to how a _real_ terminal behaves.

Constraints

  • exclusiveMinimum = 0

:rows: 5
:rows: 10

Player Configuration#

Auto-Play#

field AsciinemaPlayerSettings.autoplay: bool = False#

Set if the playback should start automatically. If True the player will start even if the video is not in the viewport! By default, the playback has to be started manually.

Possible reasons to enable auto-play is to showcase some rich help-text of a command, where the full output is shown by tuning the number of _rows_ to display. Another reason is to start an eye-catching animation at the top of the page.

:autoplay: “false”
:autoplay: “true”

Preload#

field AsciinemaPlayerSettings.preload: bool = False#

Set this option to true if the recording should be preloaded on player’s initialization.

Loop#

field AsciinemaPlayerSettings.loop: Union[bool, pydantic.types.PositiveInt] = False#

If set to True video will loop indefinitely. When set to a number, then the recording will be re-played loop times and stopped after that.

The number of loops will only be respected in the “first” playback. Please note that the recording will immediately restart once it has finished. If there is no wait-time after finishing the loop, it will look like no output has been generated since the next loop will “repaint” the window.

:loop: 3

Start-At#

field AsciinemaPlayerSettings.startat: Union[sphinx_cli_recorder.asciinema_player_settings.StrictPositiveInt, str] = 0#

Start playback at a given time. Supports the following formats:

  • 122 (number of seconds as integer)

  • “1:03” (“mm:ss”)

  • “1:02:03” (“hh:mm:ss”)

Validated by

  • startat_match

Speed#

field AsciinemaPlayerSettings.speed: pydantic.types.PositiveFloat = 1.0#

Playback speed. 2.0 means 2x faster.

Constraints

  • exclusiveMinimum = 0

Idle Time Limit#

field AsciinemaRecorderSettings.idletimelimit: pydantic.types.PositiveFloat = 5#

Limit recorded terminal inactivity to max <idletimelimit> seconds. Not milli-seconds!

Constraints

  • exclusiveMinimum = 0

Fit#

field AsciinemaPlayerSettings.fit: sphinx_cli_recorder.asciinema_player_settings.AsciinemaFit = AsciinemaFit.width#

Controls the player’s sizing behaviour inside its container element.

Can be one of:

  • “width”

  • “height”

  • “both”

  • “none”

Poster#

field AsciinemaPlayerSettings.poster: Optional[str] = None#

The preview frame to display until the playback is started. Can either be plain text encoded as: data:text/plain,Poster TXT or the display recording “frame” at a given time using the NPT (“Normal Play Time”) notation: npt:0:1.

If no prefix is given, the data:text/plain,Poster prefix is added.

Validated by

  • poster_match

:poster: “data:text/plain,rich –help”
:poster: “rich –help”
:poster: “npt:0:1”

Theme#

Define the style of the background and color palette of the terminal.

field AsciinemaPlayerSettings.theme: sphinx_cli_recorder.asciinema_player_settings.AsciinemaTheme = AsciinemaTheme.asciinema#

Select a pre-defined Asciinema theme for the terminal/text color. Can be one of:

  • “asciinema”

  • “monokai”

  • “tango”

  • “solarized-dark”

  • “solarized-light”

:theme: “asciinema”
:theme: “monokai”
:theme: “tango”
:theme: “solarized-dark”
:theme: “solarized-light”

Scripted Interaction Configuration#

Between Characters#

field SleepTimes.between_character: pydantic.types.NonNegativeFloat = 0.1#

The time to wait between each character in seconds.

By default 0.1s

Constraints

  • minimum = 0

Between Commands#

field SleepTimes.between_commands: pydantic.types.NonNegativeFloat = 1.0#

The time to wait between sending _commands_ (and before the first) in seconds.

By default 1.0s

Constraints

  • minimum = 0

Before Close#

field SleepTimes.before_close: pydantic.types.NonNegativeFloat = 1.0#

The time to wait after sending the last command in seconds. This translates to how long the final output should be shown before stopping the stream.

By default 1.0s

Constraints

  • minimum = 0

Timeout#

field SleepTimes.timeout: pydantic.types.NonNegativeFloat = 30#

The time to wait before interrupting to wait for an output from the terminal and to raise an exception. This timeout is used for both the plain and the interactive command runner.

Constraints

  • minimum = 0
